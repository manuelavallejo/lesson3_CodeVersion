Highlight #1 Git

Git is a version control system that stores data. Git is different than other version control systems because instead of storing data in the form of files, it stores data in the form of snapshots. For example, every time you make a commit in Codio, it takes a picture of your modified work. Think of Git as a system of photos that has references attached to it.

Highlight #2 Git Local Repository

The local repository is the place where you can save your work before it gets pushed to the remote repository. You can view the history of your saved work and display it without having to access it from outside server.

Highlight #3 Git Remote Repository

Remote repositories are copies of your work hosted on a network. You are able to push and pull data from a remote repo to share with other people. Remote repositories are useful in many ways, one of them being a safety net for computer crashes. If your computer experiences a crash, having backed up your work to a remote repo will ensure the recovery of those items.
